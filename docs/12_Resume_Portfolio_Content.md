## Interview answer: Tell me about this project

“ShopFlow QA Validation Suite is a full-stack e-commerce testing project. I built a small React, Express, and PostgreSQL application, then created a complete QA suite around it. I tested customer workflows like registration, login, products, cart, checkout, and order history, plus an admin workflow for updating order status. I used manual test cases, Postman, SQL queries, bug reports, smoke/regression checklists, release notes, and pytest automation.”

## What application did you test?

“I tested a demo e-commerce application called ShopFlow. It has customer-facing workflows and an admin order-status workflow, so it gave me a realistic application to test from UI, API, and database layers.”

## What types of testing did you perform?

“I performed manual, functional, UI, API, SQL/database, smoke, regression, negative, and basic automation testing.”

## How did you use Postman?

“I used Postman to validate the backend APIs for login, registration, products, cart, orders, and admin status update. I used variables for base URL, tokens, product ID, cart item ID, and order ID.”

## How did you validate data using SQL?

“I wrote SQL queries to confirm that users, cart items, orders, and order_items were correctly saved. I also checked that the order total matched the sum of item totals and that admin status updates were saved in the database.”

## What did you automate using pytest?

“I automated repeatable API tests, including health check, login, product list, product details, add to cart, checkout, order history, unauthorized access, and admin status update.”

## What bugs did you document?

“I documented realistic bugs such as incorrect cart totals, unclear duplicate email messages, empty cart checkout, order history refresh issues, and admin order status not saving correctly.”

## Why is this project relevant to QA roles?

“It is relevant because it shows I can test a real application across multiple layers: UI, API, and database. It also shows documentation skills, bug reporting, regression thinking, and beginner-level automation.”
